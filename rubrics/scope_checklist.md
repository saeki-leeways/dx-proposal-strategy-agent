# スコープ確認チェックリスト

提案時に「どこまでを今回対象にするか」を曖昧にしないための観点集です。

## 1. 対象組織
- 全社 / 特定事業部 / 特定部門 / 子会社のどこまで含むか
- 意思決定者と現場責任者は誰か
- 情報システム部門、データ部門、業務部門の役割分担は明確か

## 2. 対象業務
- どの業務プロセスを対象にするか
- As-Is 可視化までか、To-Be 設計までか
- 業務標準化、ルール整備、運用設計を含むか

## 3. 対象システム
- 対象となる基幹 / 周辺 / SaaS / DWH / BI / AI基盤は何か
- 現行改修か、新規導入か、併用か
- 外部連携、API連携、権限連携を含むか

## 4. 対象データ
- マスタ、トランザクション、ログ、外部データのどこまで含むか
- データ統合、品質改善、ガバナンス、メタデータ整備を含むか
- AI活用を前提にするなら教師データや評価データの整備まで含むか

## 5. 対象成果物
- 構想書、要件定義書、PoC報告書、ロードマップ、運用設計書など何を納品するか
- 提案段階で約束する成果物粒度は適切か

## 6. 期間・体制
- いつまでに何を出すか
- 顧客側アサイン前提は何か
- ベンダー / パートナー / 内製チームとの役割境界は明確か

## 7. 非スコープ定義
- 本番実装、データ移行、教育、運用保守、展開支援のどこまで除外するか
- 他部門展開や全社ロールアウトを今回含めない場合、明記されているか
- 法務、調達、セキュリティ審査の実施主体が明確か

## 8. 提案時の見落とし注意
- 「構想」と言いながら実装期待を持たせていないか
- PoCと本番化の責任範囲が混同されていないか
- データクレンジング負荷が暗黙に顧客側へ押し込まれていないか
- 部門横断調整コストが見積りから漏れていないか
