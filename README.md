# DXコンサル営業向け 提案戦略エージェント

顧客ヒアリング情報から、DX/AI/データ基盤案件向けの **提案戦略・提案シナリオ・スコープ・フェーズ分割・主要リスク・提案書骨子** を日本語で生成する Codex skill です。

## できること
- 顧客課題を **戦略 / 業務 / システム / データ / 組織** に分解
- 提案シナリオを **最低3案** 生成
- **スコープ / 非スコープ** を明示
- フェーズ分割と各フェーズの成果物・完了条件を整理
- 主要リスクと対応策を整理
- 不明点を **確認論点** として別出し
- 提案内容を Trello カードに落とし込むための連携手順を提示

## ディレクトリ構成
- `skills/proposal-strategy/SKILL.md`: skill 本体
- `templates/strategy_output.md`: 出力テンプレート
- `rubrics/scope_checklist.md`: スコープ検討観点
- `rubrics/risk_checklist.md`: リスク検討観点
- `inputs/sample_client_brief.md`: 入力サンプル
- `outputs/sample_strategy.md`: 出力サンプル

## 利用手順
1. 顧客ヒアリング情報を Markdown で用意します。
   - 必須に近い項目: 顧客概要、背景、課題、現状システム / データ / 組織、期待成果、制約条件
2. Codex で `proposal-strategy` skill を使って、ヒアリング情報を入力します。
3. 必要に応じて以下ファイルを参照しながら出力を調整します。
   - `templates/strategy_output.md`
   - `rubrics/scope_checklist.md`
   - `rubrics/risk_checklist.md`
4. 提案シナリオ 3 案以上、スコープ / 非スコープ、確認論点が揃っているか確認します。

## Trello連携（提案実行管理）
提案戦略の合意後、実行管理を Trello で行う場合は `docs/trello_integration.md` の手順に沿ってください。

- 事前準備（APIキー / トークン / Board / List）  
- 提案シナリオ・フェーズ分割をカードに変換するルール  
- `curl` でのカード一括作成例  
- ラベル / 期限 / 担当者の付与ルール  

## 推奨プロンプト例
```md
この顧客ヒアリング情報から、DX/AI/データ基盤案件として提案戦略を作ってください。
以下を必ず含めて、日本語で整理してください。
- 顧客課題の分解（戦略 / 業務 / システム / データ / 組織）

- スコープ / 非スコープ

```

## 注意点
- 本 skill の出力は、ヒアリング情報に基づく仮説整理です。
- 顧客の予算、体制、セキュリティ制約、既存ベンダー状況に応じて最終提案内容は調整してください。
