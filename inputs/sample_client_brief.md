# 顧客ヒアリング情報（サンプル）

## 1. 顧客概要
- 企業名: 東和流通ホールディングス
- 業種: 小売・卸売
- 売上規模: 2,000億円規模
- 事業特徴: 全国の物流拠点と店舗網を持ち、EC比率が拡大中

## 2. 相談背景
- 中期経営計画で「需要予測高度化」「在庫最適化」「データドリブン経営」を掲げている
- 一方で、店舗・EC・物流のデータが分断され、意思決定に時間がかかっている
- 既存DWHはあるが、部門ごとに独自集計が横行し、数値が一致しない

## 3. 顧客が感じている課題
- 需要予測精度が低く、欠品と過剰在庫が併発している
- 商品マスタや店舗コード体系が部門ごとに異なる
- 営業企画部はダッシュボード整備を望んでいるが、情報システム部は基盤刷新を優先したい
- AI活用への期待は高いが、学習に使える整備済みデータが不足している
- 現場では Excel による補正業務が多く、標準業務が見えにくい

## 4. 現状システム / データ / 組織
### システム
- 基幹システム: 販売管理、在庫管理、購買管理が別パッケージ
- 周辺: EC基盤、WMS、BIツール、各部門の Excel / Access
- 連携: ファイル連携中心で、リアルタイム性は低い

### データ
- 販売、在庫、購買、物流、会員、商品マスタが分散
- データ品質基準やオーナー定義が曖昧
- 分析用データモデルが標準化されていない

### 組織
- 経営企画部がスポンサー
- 営業企画部、物流部、情報システム部が主要関係者
- データ専門組織は小規模で、全社横断ガバナンス機能は未整備

## 5. 期待成果
- まずは需要予測と在庫可視化で成果を出したい
- 将来的には全社データ活用基盤を整備したい
- 6か月以内に経営層へロードマップを提示したい

## 6. 制約条件
- 初年度予算は限定的
- 本番システムの大規模刷新はすぐには難しい
- 個人情報を含む会員データの取り扱いには厳格な審査が必要
- 顧客側の専任体制は薄く、兼務メンバー中心
